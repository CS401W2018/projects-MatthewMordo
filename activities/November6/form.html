The form tag has an "action" attribute. What does it do?

  • The action attribute specifies the URL where the form data will be sent when the form is submitted. This is typically a script (like a PHP, Python, or JavaScript file) or a server-side application endpoint that will process the form data.

The form tag has a "method" attribute. What does it do?

  • The method attribute defines the HTTP method used when submitting the form data to the server. The two most common methods are GET and POST.

What is the difference between a "POST" and a "GET"?

  • Get: The GET method appends the form data to the URL as query parameters. The data is visible in the URL, so it is less secure.
  • GET is generally used for retrieving data (e.g., search queries) and should not be used for sensitive information like passwords.
  • GET requests have a limit on the amount of data that can be sent due to URL length restrictions.
  • The POST method sends the form data in the body of the request rather than the URL. As a result, the data is not visible in the URL and can be much larger.
  • POST is used for submitting data that modifies the server state, such as creating or updating records, or for handling sensitive information.
